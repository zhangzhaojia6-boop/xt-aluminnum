# 正式身份入口准备清单（保留原企业微信文件名）

## 目标

把当前内部系统的正式入口从本地 `localhost` 切到生产环境可访问入口。

当前 `Phase 1` 的执行口径是：

1. 浏览器可直接访问 `/mobile` 与 `/review/factory`
2. 钉钉 H5 / 免登入口优先
3. 企业微信不是本轮快速试跑和内部试点的上线前提

说明：本文件名沿用 parse15 阶段产物，方便和旧计划互相对应；但实际执行口径已经切到“正式身份入口”而不是“企业微信必上”。

## 系统侧固定信息

### 用户入口

1. 手机填报入口：`https://<你的正式域名>/mobile`
2. 审阅入口：`https://<你的正式域名>/review/factory`

### 当前已挂载的身份入口

1. 当前主入口：`POST /api/v1/dingtalk/login`
2. 浏览器账号口令入口：`POST /api/v1/auth/login`

### 兼容说明

1. `backend/app/routers/wecom.py` 仍保留兼容实现
2. 但当前 `backend/app/main.py` 未挂载 `/api/v1/wecom/login`
3. 如果后续要恢复企业微信正式入口，需要先重新挂载路由，再补企业微信后台配置

## 生产环境变量

如果本轮只走浏览器访问链路，至少需要保证基础生产配置完整：

```env
APP_ENV=production
SECRET_KEY=
INIT_ADMIN_PASSWORD=
VITE_API_BASE_URL=/api/v1
```

如果要启用钉钉正式入口，再补：

```env
DINGTALK_CORP_ID=
DINGTALK_APP_KEY=
DINGTALK_APP_SECRET=
DINGTALK_AGENT_ID=
```

如果后续要恢复企业微信正式入口，才需要：

```env
WORKFLOW_ENABLED=true
WECOM_APP_ENABLED=true
WECOM_CORP_ID=
WECOM_AGENT_ID=
WECOM_APP_SECRET=
```

如果要同时启用群消息推送，再补：

```env
WECOM_BOT_ENABLED=true
WECOM_BOT_DRY_RUN=false
WECOM_BOT_WEBHOOK_URL=
```

## 上线前必须确认

1. 正式域名已经能通过 HTTPS 访问
2. 真实身份账号和系统账号已经完成映射
3. 管理员、主操、专项 owner 至少各有 1 个真实可登录账号
4. 浏览器访问 `/mobile` 与 `/review/factory` 已通过
5. `readyz` 返回 `hard_gate_passed=true`

## 钉钉 / 企业微信后台需要现场确认的事项

由于这部分需要真实平台管理后台权限，当前仓库只能先把系统侧准备好；上线时请按实际选用的入口完成：

1. 应用可见范围
2. 正式访问域名相关配置
3. H5 入口地址配置
4. 若启用企业微信 JS-SDK，再补对应域名配置
