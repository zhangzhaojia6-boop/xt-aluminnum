# 快速试跑运维手册

这份手册只服务当前 `Phase 1 一个车间快速试跑`。

目标不是做复杂运维，而是让常见动作都能按固定命令完成。

## 1. 首次部署

在云主机项目目录执行：

```bash
./scripts/deploy_trial.sh
```

它会做：

1. 检查 compose 配置
2. 启动 `db / backend / nginx`
3. 输出当前容器状态

## 2. 健康检查

部署后执行：

```bash
./scripts/check_trial_stack.sh https://你的域名
```

它会检查：

1. `/healthz`
2. `/readyz`
3. 当前 compose 服务状态

## 3. 更新代码

推荐通过 GitHub 更新：

```bash
git pull
./scripts/deploy_trial.sh
```

## 4. 查看日志

后端日志：

```bash
docker compose -f docker-compose.yml -f docker-compose.prod.yml logs backend --tail 200
```

Nginx 日志：

```bash
docker compose -f docker-compose.yml -f docker-compose.prod.yml logs nginx --tail 200
```

## 5. 备份数据库

```bash
./scripts/backup_db.sh
```

备份文件默认会放到 `backups/`。

## 6. 恢复校验

```bash
./scripts/restore_db.sh backups/你的备份文件.dump
```

这个动作会把备份恢复到一个校验库，确认备份不是坏的。

## 7. 回滚思路

如果新版本试跑异常：

1. 切回上一个 Git 提交
2. 重新执行部署脚本
3. 若怀疑数据库问题，优先先做恢复校验，不直接覆盖生产库

## 8. 本轮不要做的事

1. 不要直接全厂放量
2. 不要在试跑时同时接企业微信正式入口
3. 不要把 `.env`、证书和密钥提交到 GitHub
