# Go-Live Cloud Rollout Runtime Plan

**Goal:** 在不扩新范围的前提下，把当前项目从“本地稳定基线”推进到“内部试点可上云试跑”。

## 当前判断

按当前仓库与验证基线：

1. `2-4 天` 可完成上云试跑
2. `5-7 天` 可进入内部正式试点
3. `10-15 天` 可达到相对稳定的现场落地状态

## Day 0：冻结发布范围

**目标：** 停止继续扩功能，收敛到一个可发布基线。

**任务：**

- [ ] 确认本周只做上线相关收口，不再继续扩 MES / 成本 / 排产
- [ ] 清点当前工作区改动并整理成“发布包含 / 暂不包含”
- [ ] 固定本轮发布口径：Phase 1 内部试点，不依赖 MES

**完成标准：**

- [ ] 有一版明确的发布边界
- [ ] 管理层、管理员、填报端口径一致

## Day 1：生产环境与部署资产

**目标：** 把云主机、域名、证书、环境变量准备好。

**任务：**

- [ ] 确认云主机（Linux + Docker）
- [ ] 确认生产域名
- [ ] 准备 HTTPS 证书
- [ ] 复制 `.env.example` 生成生产 `.env`
- [ ] 填写正式数据库口令、`SECRET_KEY`、管理员账号

**完成标准：**

- [ ] `docker compose -f docker-compose.yml -f docker-compose.prod.yml config` 通过
- [ ] 生产 `.env` 填写完成

## Day 2：代码发布与云端起服务

**目标：** 在云端把服务真正拉起来。

**任务：**

- [ ] 上传或拉取代码到云主机
- [ ] 放置证书到 `ssl/`
- [ ] 运行生产 compose
- [ ] 检查 `db / backend / nginx`
- [ ] 检查 `/healthz` 与 `/readyz`

**完成标准：**

- [ ] `docker compose ps` 正常
- [ ] `/readyz` 返回 `hard_gate_passed=true`

## Day 3：现场主数据与账号过数

**目标：** 让现场第一批人真的能进系统。

**任务：**

- [ ] 核对车间、班次、机台、模板
- [ ] 核对主操账号
- [ ] 核对专项 owner 账号
- [ ] 核对管理员账号
- [ ] 跑一遍试点排班与 readiness

**完成标准：**

- [ ] 试点人员能登录
- [ ] 模板与岗位字段匹配现场口径

## Day 4：内部试跑 UAT

**目标：** 跑一轮最小真实链路。

**任务：**

- [ ] 主操提交 1 条正常数据
- [ ] 主操提交 1 条异常数据
- [ ] 专项 owner 提交 1 条补录
- [ ] 管理员在审阅端确认来源、状态与风险
- [ ] 确认日报 / 驾驶舱可见

**完成标准：**

- [ ] 填报、退回、汇总、审阅四条主链路可用
- [ ] 没有权限串看、串改问题

## Day 5-7：内部正式试点

**目标：** 扩到 1 个车间或 2-3 个岗位真实使用。

**任务：**

- [ ] 选 1 个真实试点班次
- [ ] 每天检查 `reporting_rate / return_rate / hard_gate_passed`
- [ ] 记录现场反馈
- [ ] 收一轮字段和文案修正

**完成标准：**

- [ ] 连续 2-3 天主链路稳定
- [ ] 没有阻断级问题

## 当前上线阻塞清单

- [ ] 发布基线尚未冻结
- [ ] 生产域名尚未确认
- [ ] 正式证书尚未就位
- [ ] 生产 `.env` 尚未生成
- [ ] 企业微信正式可信地址 / 回调地址未确认
- [ ] 现场主数据尚未最终过数
- [ ] 真实 UAT 尚未完成

## 执行优先级

1. 先冻结发布范围
2. 再准备云环境
3. 再上云拉起服务
4. 再做现场过数和 UAT

## 本轮立即执行项

1. 补齐 `docs/部署文档.md`
2. 固化当前倒排表
3. 下一步开始做“发布基线清点”
