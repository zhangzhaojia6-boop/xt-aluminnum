# Review Home AI Assistant Redesign Design

## 背景

当前审阅端虽然已经具备：

- `今日总览`
- `流程追踪`
- `来源泳道`
- `风险提示`

但管理端首页仍然存在 4 个核心问题：

1. 信息层级偏重，首页还是像“后台页拼装”而不是“指挥面”
2. 文字说明偏多，不够一眼看懂
3. 三端打开后的体验差异没有被明确设计，电脑/平板/手机容易出现“能看但不顺”
4. `AI 助手` 还没有被放进首页主结构，只停留在概念层，没有清楚预留 `问答 / 搜索 / 检索 / 图像生成 / MCP / 数据库 / 自动化`

用户已明确选择：

- 首页主方向：`今日总览 + 流程追踪`
- 助手方向：`B = 轻常驻 + 深展开`
- 适配方向：`填报端手机优先 + 审阅端电脑优先 + 平板居中兼容`
- 品牌气质：`现代东方感`
- 品牌主视觉：`铝材流线`
- logo 结构：`图形 + 鑫泰铝业全称`
- logo 方向：`B = 铝带飞白型`
- 主色方向：`青蓝银白`

## 本轮目标

把厂长/管理员审阅首页升级成真正的“指挥首页”，并同时为 AI 助手与后续 API 联通做好壳层和测试接缝。

本轮具体目标：

1. 审阅首页首屏重构为 `双核指挥型`
2. 小机器人流程追踪升级成更智能、更美观的运行图层
3. 首页显式预留 `AI 助手层`
4. 三端自适应规则正式落地
5. 为后续接真实 API / MCP / 数据库动作准备稳定前后端 contract 与测试入口

## 设计原则

- 多图形，少文字
- 首页优先看懂“今天怎么样”，其次看懂“数据怎么来的”
- AI 不是大段解释文案，而是明确的能力入口与工作台
- 审阅端电脑优先，但手机和平板不能看起来像“缩小版网页”
- 不在这一轮伪造真实 LLM / MCP / 数据库能力，只预留稳定接缝
- 不扩 MES / 成本 / 排产业务范围
- 视觉上保持现代东方感，不能做成厚重政务后台或幼态卡通页
- 图形语汇优先复用“铝带飞白”的流线、留白和掠光，而不是额外堆装饰

## 信息架构

### 1. 首页第一层：今日总览

首屏顶部先用 4-6 个高识别度图形卡片承载：

- 今日产量
- 缺报
- 异常
- 交付风险
- 单吨能耗
- 可选的库存 / 合同进度

要求：

- 大数字优先
- 解释性文字压到最短
- 风险状态用颜色、图形、轻量标签表达，不用长句堆叠

### 2. 首页第二层：流程追踪

流程追踪继续保留，但不再是“单独一块说明卡”，而是升级成首页第二主视觉：

- `主操直录`
- `专项补录`
- `系统汇总`
- `结果发布`

每个阶段用更柔和、更聪明的机器人/执行单元表达，并通过轻量动画呈现：

- 数据正在流动
- 当前卡在哪
- 哪个阶段健康、预警、空闲

重点不是卖萌，而是让管理层在 3 秒内明白：

- 数据由谁送来
- 已经处理到哪
- 哪一步出问题

### 3. 首页第三层：AI 助手入口

助手采用 `轻常驻 + 深展开`：

- 首页不放完整聊天面板
- 首页放一个清晰但轻量的 AI 助手条
- 助手条直接暴露 4 个高频快捷动作

建议首批快捷动作：

- 查今天异常
- 搜某卷/某班次/某车间
- 生成日报图
- 触发自动化

用户展开后，再进入完整的 AI 工作台。

## AI 助手层设计

### 首页轻常驻层

首页轻常驻层只负责两件事：

1. 让用户知道系统已经具备 AI 助手能力
2. 让用户用最短路径进入高频动作

首页轻常驻层不负责：

- 长对话
- 复杂历史
- 深度结果展示

### 深展开工作台

展开后的 AI 助手工作台按能力分为 6 个区：

1. `问答`
   - 例如：今天产量为什么少了 12 吨
   - 例如：哪个车间现在最危险

2. `搜索`
   - 跨全站搜人、卷、班次、车间、合同、异常

3. `检索`
   - 基于 dashboard / history_digest / runtime_trace / lanes / 文档沉淀做结构化检索

4. `图像生成`
   - 生成趋势图
   - 生成异常说明图
   - 生成日报视觉卡片

5. `MCP / 数据库连接`
   - 预留后续 MCP 工具入口
   - 预留数据库查询与工具调用结果展示位

6. `自动化动作`
   - 催报
   - 生成摘要
   - 推送
   - 分派
   - 留痕

## 三端自适应设计

### 填报端

- 手机优先
- 固定使用窄宽居中工作台
- 电脑打开时保持“移动工作台居中”，不做桌面化扩展

### 审阅端

- 电脑优先
- 宽屏下首页采用 `双核首屏 + 下层明细`
- 平板下从多列降到双列
- 手机下改成纵向摘要优先，流程追踪折叠或横滑，不直接硬塞桌面栅格

### 平板兼容

平板是独立兼容层，不是简单缩小电脑端：

- 首页指标卡减少同屏数量
- 流程追踪从横向四列改成两列或横滑
- AI 助手从轻常驻条切成抽屉入口或底部面板

## 页面范围

本轮只重构：

- `厂长/管理员审阅首页`
- 相关审阅壳层
- AI 助手首页入口与深展开壳层
- 三端响应式布局规则
- 首页品牌化表达，包括 logo/字标占位、品牌色、标题艺术字和统一图形语汇

本轮不重构：

- 全部主数据后台页面
- 填报业务流程逻辑
- 真实 AI 对话能力
- 真实 MCP 工具链
- 真实数据库写动作

## API 测试准备

这一轮需要同时准备后续联通 API 的测试接缝，但不要求真实接入第三方模型或自动化系统。

建议做法：

1. 为 AI 助手定义稳定前端 API 模块
2. 为后端定义可测试的占位路由与 schema
3. 用 mock / deterministic response 跑通前端壳层与交互测试
4. 用 router tests 锁定后端 contract

建议首批 API 面：

- `POST /api/v1/assistant/query`
  - 问答 / 搜索 / 检索统一入口
- `POST /api/v1/assistant/generate-image`
  - 图像生成占位入口
- `GET /api/v1/assistant/capabilities`
  - 当前能力和连接状态

这些接口首轮只返回 deterministic mock 数据，目的是：

- 让前端壳层可开发、可验收
- 让后续真实接 LLM / MCP / 数据库时不推翻 UI

## 品牌与视觉锚点

### Logo 方向

当前品牌方向固定为 `图形 + 鑫泰铝业全称`，图形采用 `铝带飞白型`：

- 主体是一条被拉伸、翻卷、留白切开的铝材流线
- 要有“金属带材高速拉出”的轻速度感和工业精度感
- 留白边缘允许带一点书法飞白意味，但整体仍保持现代几何秩序
- 不做传统印章感，不做卡通 mascot 风

### 色彩方向

品牌主色固定为 `青蓝银白`：

- 青蓝负责科技感、清洁感和主品牌识别
- 银白负责金属感、专业感和空间留白
- 可用少量深海军蓝压住层次，但整体不能发灰或过暗

### 首页视觉表达

厂长/管理员首页的视觉表达需要和品牌同源：

- hero 区背景、流程追踪路径、AI 助手展开态都复用“铝带飞白”的流动语汇
- 机器人与执行单元更像智能、柔和、轻量的工业助手，而不是幼态玩偶
- 艺术字只用于标题级别，正文仍以清晰、专业、易扫读为第一优先

## 验收标准

- 审阅首页首屏已收成 `双核指挥型`
- 首页文字明显减少，图形比例明显提高
- 小机器人流程追踪更智能、更美观
- 首页已显式存在 `AI 助手` 入口
- 助手展开层已预留 `问答 / 搜索 / 检索 / 图像生成 / MCP / 数据库 / 自动化`
- 填报端、审阅端、平板兼容规则清晰且可验证
- API 接缝有明确 contract，可做前后端测试

## 本阶段不做

- 不接真实大模型 API
- 不接真实 MCP 生产工具
- 不接真实数据库写动作
- 不把首页做成新的复杂后台
- 不把 AI 助手做成满屏聊天框
