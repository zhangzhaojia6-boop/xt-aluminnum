# 发布冻结基线清单

## 适用阶段

用于当前 `Phase 1 内部试点` 的发布冻结。

当前冻结目标不是“功能全做完”，而是先冻结一版能真实上云试跑、能给现场填报和管理员审阅使用的稳定基线。

## 本次冻结范围

### 包含

1. `双端分离`
   - `/entry` 只给主操与专项 owner 填报
   - `/manage/*` 只给管理员与管理层审阅
   - `/mobile`、`/review/*` 仅作历史兼容入口
2. `填报端收口`
   - 主操一岗一表
   - 班末确认字段单独拆分
   - 移动端文案减字、去后台腔
3. `审阅端收口`
   - 来源泳道
   - 来源标签与结果卡映射
   - `analysis_handoff` typed contract
4. `AI 审阅交接包`
   - 已包含 `readiness / blocking_reasons / priority / attention_flags`
   - 已包含 `trend / freshness / data_gaps`
   - 已包含 `section_matrix / section_reasons / source_matrix / source_variants / action_matrix`

### 不包含

1. `MES` 正式联调
2. 扫码自动补数正式启用
3. 成本核算自动化
4. 排产建议自动化
5. 合同排序自动化
6. 企业微信正式消息大规模推送

## 当前冻结验证基线

以 `2026-04-20` 本地最新验证为准：

1. 后端全量：`docker compose run --rm backend sh -lc "pytest -q"` → `387 passed`
2. 前端构建：`cd frontend && npm run build` → 通过
3. 关键 E2E：
   - `mobile-entry-smoke`
   - `review-runtime`
   - `dynamic-entry-layout`
   - `zd1-machine-smoke`
   → `4 passed`
4. 就绪检查：`https://localhost/readyz` → `hard_gate_passed=true`

## 冻结前最后确认

1. 不再继续扩 `MES / 成本 / 排产 / 经营自动化`
2. 不再新增新的前端大页面
3. 只允许修：
   - 阻断级 bug
   - 上云必需配置
   - 正式身份入口必需配置（浏览器 / 钉钉）
   - UAT 暴露出的现场阻断问题
4. `git status --short` 不再包含测试缓存、临时截图和误生成目录
5. GitHub 远端已接入，云主机可直接拉代码

## 冻结后立即执行

1. 准备生产 `.env`
2. 准备云主机、域名和证书
3. 完成正式身份入口配置（当前优先浏览器 / 钉钉，企业微信不是 Phase 1 前提）
4. 跑现场 UAT
