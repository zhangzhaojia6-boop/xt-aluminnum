# 供应商对接手册（前端重构版）

## 1. 项目定位

本系统为铝厂现场生产协同系统，前端分为三类面：

- Entry（独立填报端）
- Review（审阅与运营端）
- Desktop Legacy（配置与兼容入口）

## 2. 关键入口

- 登录：`/login`
- 独立填报端：`/entry`
- 审阅端：`/review/overview`
- 厂级看板：`/review/factory`
- 车间看板：`/review/workshop`

兼容入口：

- `/mobile/*` 保留为兼容并转发至 `/entry/*`
- `/dashboard/*` 保留兼容 redirect

## 3. 权限模型（前端）

基于 [auth.js](/D:/zzj Claude code/aluminum-bypass/frontend/src/stores/auth.js)：

- `canAccessFillSurface`
- `canAccessReviewSurface`
- `canAccessDesktopConfig`
- `isAdmin / isManager / isReviewer / isMobileUser`

路由守卫位于 [index.js](/D:/zzj Claude code/aluminum-bypass/frontend/src/router/index.js) 的 `beforeEach`。

## 4. API 对接范围

前端默认兼容以下接口族：

- `/api/v1/auth/*`
- `/api/v1/mobile/*`
- `/api/v1/attendance/*`
- `/api/v1/work-orders/*`
- `/api/v1/dashboard/*`
- `/api/v1/reports*`
- `/api/v1/realtime/*`
- `/api/v1/master/*`
- `/api/v1/imports/*`
- `/api/v1/users/*`
- `/api/v1/mes/*`
- `/api/v1/reconciliation/*`
- `/api/v1/quality/*`
- `/api/v1/assistant/*`
- `/healthz` `/readyz`

## 5. 成本策略引擎对接

前端成本计算模块位于 `frontend/src/services/costing/*`，支持策略：

- `CASTING_MACHINE_LABOR_SPLIT`
- `FINISHING_PARALLEL_PROCESS`
- `HOT_ROLLING_SHIFT_DAILY`
- `TENSION_LEVELING_MAIN_PLUS_AUX`
- `LOSS_DUAL_CALIBER`

后端若提供专用结果接口，建议按 `strategy_code + workshop + date` 返回：

- 总成本
- 产量吨耗
- 通货吨耗
- 工序分解
- 价格快照
- 校差记录

## 6. AI 对接

- 能力入口：`/api/v1/assistant/capabilities`
- 问答：`/api/v1/assistant/query`
- 出图：`/api/v1/assistant/generate-image`
- live probe：`/api/v1/assistant/live-probe`

前端会在接口失败时退回 mock fallback，以保证 UI 不中断。

## 7. 联调建议

1. 先联通登录、`/review/factory` 与 `/entry`
2. 再联通数据接入、审阅任务、成本中心
3. 最后联通 AI live probe 与运维指标

## 8. 交付验收建议

1. 跑 `npm --prefix frontend run build`
2. 验证关键路由和权限边界
3. 运行关键 e2e 子集
4. 输出缺口到 `docs/known-gaps-and-todos.md`
