## 真实旧口径里已有，但当前系统未完整吸收的面

1. 综合日报不仅有产量，还同时承载：
- 投料量
- 日/月累计产量
- 废料
- 月成品率
- 电耗 / 气耗
- 备料 / 入库 / 发货 / 接合同量

2. 成品率并不是单行指标，而是多车间、多级表头矩阵：
- 1450 / 1650+2050 / 1850 / 拉矫 / 精整 / 园区飞剪 / 公司
- 同时含 M/P 指标与公司总成品率

3. 合同报表是“按日 sheet”组织，不是单张平表：
- 每天一个 sheet
- 同时展示当日合同、月累计合同、投料和坯总量

4. 发货表目前仍是图片事实，不是结构化数据：
- 入库 / 月入库 / 日出库 / 月出库
- 新厂卷板 / 涂板
- 总库存
- 车间主任签字

## 对当前系统的结论

- 当前系统已经覆盖“移动填报 -> 校验 -> 汇总 -> 发布 -> 触达”主链路。
- 但旧总统计现实世界里还有 3 类 sidecar 信息面没有被结构化吸收：
  - 合同口径
  - 发货 / 库存口径
  - 成品率矩阵口径
- 因此当前优化方向不应是把网站重新做回人工总统计台，而应是：
  - 主链路继续坚持 workflow-first
  - 旧统计口径沉淀为 observer sidecar / legacy profile / import normalization
