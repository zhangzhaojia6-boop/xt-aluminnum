# 异常处理最小SOP（试点版）

## 1. 每班查看异常
```bash
cd backend
python scripts/check_pilot_anomalies.py --date 2026-04-06 --json
```

## 2. 按异常类型处理
- 产出大于投入：优先核对本班投入/产出数据，修正后重新提交。
- 班次缺报：立即联系当班负责人补报；补报后再次运行异常检查确认清除。
- 能耗异常波动：先核对设备状态与能耗抄录，再确认是否为工艺波动。
- 出勤异常：核对排班人数与实际到岗，修正出勤或备注说明。
- 跨班次跳变：核对班次交接、原料批次、设备状态，确认是否真实波动。

## 3. 通知与升级
- 日报推送已包含异常摘要，管理层会收到异常条数与摘要。
- 如需暂停外发但保留填报：
```env
AUTO_PUSH_ENABLED=false
```

## 4. 关闭条件
- 当日异常总数持续下降。
- 同类异常连续两班不再重复出现。
- 关键异常（产出大于投入、班次缺报）清零后再恢复正常节奏。


## 5. 放量前置条件
- 异常 SOP 只在 `docs/pilot-readiness-checklist.md` 三个 Gate 全通过后进入日常执行。
