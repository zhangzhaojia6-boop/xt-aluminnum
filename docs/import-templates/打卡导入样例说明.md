# 打卡导入样例说明

样例文件: `docs/import-templates/clock_sample.csv`

字段说明:
- `employee_no`: 员工工号，优先用于匹配员工
- `clock_datetime`: 打卡时间，推荐格式 `YYYY-MM-DD HH:mm:ss`
- `clock_type`: 打卡类型，支持 `in/out`（也兼容 `上班/下班`）
- `dingtalk_record_id`: 钉钉打卡记录唯一标识（可选但强烈建议，避免重复导入）
- `device_id`: 设备标识，参与去重唯一键
- `location_name`: 打卡地点（可选）

最小成功条件:
- 文件格式为 `csv` 或 `xlsx`
- 至少包含上述字段
- `employee_no` 在员工主数据中存在
- 同一 `dingtalk_record_id` 不重复
- 同一 `(employee_id, clock_datetime, clock_type, device_id)` 不重复
